=== WRITING GUIDELINES ===
- **Experience & Projects**:
  - Focus on WHAT was done, HOW it was done (technologies), and WHY it mattered (impact).
  - Avoid passive voice (e.g., "Responsible for...") -> Use active voice (e.g., "Directed...", "Built...").
  - If a GitHub repository has a README, extract key features and technical challenges to populate 'highlights'.
- **Summary**:
  - Synthesize the candidate's core strengths (e.g., "Full Stack Developer with expertise in React and Node.js...").

=== DATA HANDLING ===
1. Standardize tech names: React.js -> React, Node -> Node.js, python -> Python.
2. Select the most impressive 4-8 projects if many are available, prioritizing those with code, stars, or deployed links.
